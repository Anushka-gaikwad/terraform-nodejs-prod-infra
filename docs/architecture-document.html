1. Architecture Overview

This document outlines the production-grade infrastructure for hosting a Node.js REST API on AWS EC2. The architecture is designed for high availability, auto scaling, security hardening, and full observability using Terraform IaC.

Design Principles Multi-AZ deployment for fault tolerance • Zero-downtime rolling deployments • Least-privilege security • Centralized logging • Infrastructure as Code (no manual console clicks)

Key Decisions & Trade-offs
  • NAT Gateway per AZ instead of single NAT — higher cost but eliminates cross-AZ single point of failure
  • SSM Session Manager instead of SSH — no port 22 exposed, auditable sessions, no key management
  • Instance Refresh for deployments — zero-downtime rolling updates without external CI/CD dependency
  • KMS CMK instead of AWS-managed keys — centralized key management, rotation, and audit via CloudTrail
  • Separate tfvars per environment — same modules for staging/production, only config differs
